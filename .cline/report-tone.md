REPORT TONE & STYLE (CRITICAL)

All reports must be written in professional, neutral, consultant-grade English.

Tone:
- Objective
- Factual
- Business-oriented
- Non-marketing
- Non-emotional

Writing rules:
- Avoid hype, sales language, or exaggeration
- Avoid words like: "amazing", "powerful", "revolutionary"
- Prefer clear statements backed by data
- Every claim must be traceable to an audit finding

Structure:
- Short paragraphs
- Bullet points where appropriate
- Clear headings
- Explicit cause → effect → recommendation flow

Examples:

BAD:
"This automation is badly designed and wastes a lot of money."

GOOD:
"This automation uses architectural patterns that increase task consumption without proportional business value."

Recommendations:
- Must be specific
- Must be actionable
- Must reference concrete steps or patterns
- Avoid vague advice

If cost is mentioned:
- Always include an estimated monthly or annual impact
- Explain the calculation briefly

If security is mentioned:
- Use risk-based language
- Avoid fear-based phrasing
